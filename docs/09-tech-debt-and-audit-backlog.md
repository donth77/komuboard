## Renderer (post-ADR-0009)

New debt created or exposed by the DOM-unification migration. Phases 0–3 shipped per-object stacking; Phase 4 perf began with **viewport culling (R1, shipped 2026-06-22)** — see [ADR-0009 §Implementation phases](./adr/0009-render-model-dom-unification.md).

| ID  | Item                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                | Sev            | Location                          | Timing                  |
| --- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------- | --------------------------------- | ----------------------- |
| R1  | ~~**No viewport culling.**~~ **Done (2026-06-22):** only objects whose world AABB intersects the viewport (+ a half-viewport margin) are mounted; `recull()` runs on every camera change, an exempt set keeps edited/actively-animated objects mounted, and selection/hit-test/⌘A read the doc not the DOM. Measured 1000-obj zoom: **435 ms → 1.5 ms**; scales past 5000 (was dead at 2000). Covered by `e2e/culling.spec.ts`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     | ✅             | `text-layer.ts` `render`/`recull` | ✅ done                 |
| R2  | **Camera is not a single container transform.** Each object's screen position is rewritten individually per frame rather than moving one transformed container (ADR perf-plan #3). Defensible (keeps contenteditable text crisp/unscaled), and **culling made it a non-issue at current scale** — the per-frame reposition loop now runs over the mounted (visible) set only. Revisit only if profiling shows the reposition loop itself is the wall.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               | Low            | `text-layer.ts:syncTransform`     | 🅿️ (culling covered it) |
| R3  | **No LOD at far zoom — deliberately deferred (2026-06-22).** Culling (R1) covers zoomed-in/normal use; it does _not_ help the one case where you zoom all the way out so thousands of objects are on screen at once (then visible ≈ total, and the un-culled cost returns). The fix is LOD — flatten dense ink/objects to one background `<canvas>` below a zoom threshold, where individual objects aren't editable anyway. **Decision: not now.** Rationale: it's a rare interaction, no board today is dense enough to hit it (M2 persistence isn't built, so thousands-of-object boards can't even persist yet), and a second render path + zoom-switch with hysteresis is the highest-risk renderer change left — premature without a profiled real-board case. **Revisit trigger:** real boards with thousands of objects _and_ telemetry/reports showing users zoom far out and hit jank. (A cheaper interim cap — e.g. a "too many objects to render at this zoom" hint, or raising the min-zoom — could precede full LOD.) | Med (deferred) | `text-layer.ts`                   | 🅿️ (trigger above)      |
| R4  | **a11y pass pending for the new DOM objects** (focus order, roles/labels on the SVG ink + stamp boxes, keyboard transform). Was easy on Konva-as-one-canvas; now each object is real DOM and should be reachable/labeled.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           | Med            | `text-layer.ts`, `styles.css`     | ⏭ (Phase 4)            |
| R5  | **`text-layer.ts` is the new god-module (~3.7k lines)** — it absorbed object render + hit-test + transform chrome + remote glide for all six types. Supersedes O2's "canvas.ts god-class" framing. Seam candidates: ink (SVG) rendering, the resize/rotate chrome, the remote-presence glide.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       | Med            | `text-layer.ts`                   | ⏭                      |
| R6  | **Client bundle is a single ~650 kB chunk** (gzip ~193 kB) — Konva + Yjs dominate. Vite warns >500 kB.
